# 研究生工位打卡与时间管理系统

一个面向研究生、科研工作者和个人工位管理场景的轻量级打卡与时间管理系统。

本项目默认仍是纯前端静态应用：无需后端、无需数据库，日常使用时直接打开 HTML 文件即可。数据保存在浏览器 `localStorage` 中，适合个人本地长期使用。

同时项目已提供 Electron 桌面版入口，用于更可靠的专注完成提醒、打卡提醒、久坐提醒、系统通知、窗口置顶、任务栏闪烁和托盘后台运行。

## 功能特性

- `打卡记录`：支持上午、下午、晚上三段上班/下班打卡，并自动判断是否合格。
- `手动补卡`：支持忘记打卡后补录指定工作日、时段、类型和实际时间，并按规则重新判定合格状态。
- `规则设置`：支持配置可打卡时间窗、合格时间窗和工作日翻页时刻，保存后立即重算展示。
- `专注时长`：支持自定义倒计时、完成提醒、每日专注统计和猫粮兑换。
- `休息计时`：支持自定义休息倒计时、暂停继续、提前结束记录和休息结束提醒。
- `猫咪陪伴`：根据专注时长兑换猫粮，支持喂食、好感度、表情、台词和点击互动。
- `久坐提醒`：支持 45 分钟坐下 + 5 分钟站立的自动循环，并保存每日站立统计。
- `任务管理`：支持任务开始/结束和任务时长统计。
- `手机克制`：快速记录克制次数，并支持成就反馈。
- `请假记录`：记录请假日期和原因，请假日会影响打卡状态。
- `统计分析`：查看打卡率、时段打卡、任务时长、手机克制和专注时长趋势。

## 浏览器版与桌面版

如果只是记录打卡、任务、专注和统计，可以继续直接打开 HTML 使用浏览器版。浏览器版不需要安装依赖，适合轻量本地使用。

如果你更依赖提醒可靠性，建议使用 Electron 桌面版。桌面版把专注完成、打卡和久坐提醒交给 Electron 主进程处理，页面最小化或窗口被遮挡时，也能通过系统通知、窗口拉起、任务栏闪烁和托盘状态提高可靠性。

| 使用方式 | 适合场景 | 说明 |
| --- | --- | --- |
| 浏览器版 | 轻量记录、直接打开 | 双击 HTML 即可使用，提醒能力受浏览器限制 |
| 桌面版 | 长时间专注、需要强提醒 | 支持 Win11 系统通知、按钮操作、托盘后台、窗口置顶和任务栏闪烁 |

## 桌面版运行

如果需要更可靠的专注提醒，可以使用 Electron 桌面版：

```bash
npm install
npm run desktop
```

桌面版能力：

- 专注开始后由 Electron 主进程调度结束提醒。
- 休息计时结束后由 Electron 主进程调度结束提醒，并支持系统通知按钮快速回到学习或再休息 5 分钟。
- 打卡提醒会根据当前规则自动调度，默认在合格截止前 10 分钟提醒。
- 久坐提醒会由主进程接管坐下 45 分钟和站立 5 分钟的阶段切换提醒。
- 到点后触发 Windows 系统通知；关键提醒会显示主窗口、窗口短暂置顶和任务栏闪烁。
- 关闭窗口时默认隐藏到托盘，专注提醒仍继续运行。
- 系统通知只在到点时弹出一次；如果用户不点击，不会连续弹出多个右下角提示。
- 打卡通知中的 `5分钟后` 属于用户主动选择的稍后提醒，会在 5 分钟后再弹出一次。
- 系统通知支持按钮操作，例如打开对应页面、稍后提醒或确认提醒。
- 托盘菜单支持打开主界面、查看提醒状态、确认提醒、清除打卡提醒和退出程序。

桌面版使用流程：

1. 进入左侧 `专注时长` 页面。
2. 设置本次专注倒计时时间。
3. 点击开始专注。
4. 时间到后，根据桌面提醒完成确认。
5. 如需真正退出程序，请右键托盘图标选择退出；直接关闭窗口会隐藏到托盘。

打包 Windows 安装包：

```bash
npm run build:win
```

打包完成后会在 `dist/` 目录生成：

- `研究生工位打卡与时间管理系统 Setup x.x.x.exe`：安装版。
- `win-unpacked/`：未压缩的 Windows 应用目录，可直接运行其中的 exe。

如果确实需要便携版，可以单独执行：

```bash
npm run build:win:portable
```

## 技术说明

- `HTML + CSS + JavaScript`：纯前端静态运行。
- `localStorage`：所有数据保存在当前浏览器本地。
- `Tailwind CSS`：样式已编译到 `assets/app.css`，不再依赖运行时 Tailwind CDN。
- `Chart.js`：统计页首次打开时按需加载，避免影响普通打卡操作。
- `Font Awesome`：用于基础图标显示。
- `Electron`：可选桌面壳，用于系统级专注提醒、托盘和窗口控制。

如果后续修改 Tailwind 源样式，可重新生成 CSS：

```bash
npx tailwindcss -c ./tailwind.config.cjs -i ./assets/tailwind-input.css -o ./assets/app.css --minify
```

## 性能优化

当前版本已经从单 HTML 重构为静态多文件结构，并针对点击响应做了优化：

- 主要业务逻辑迁移到 `assets/app.js`，页面入口更轻。
- 样式迁移到本地静态 CSS，移除 Tailwind 运行时编译开销。
- 点击保存时优先写入主快照，旧版兼容 key 延迟到空闲时写入。
- 高频 DOM 节点使用缓存读取，减少重复查询。
- 专注、任务、久坐倒计时合并为统一秒级 tick，无计时时不运行秒级循环。
- 统计图表只在统计页可见时刷新，Chart 实例复用，不再每次销毁重建。

## 数据存储

系统使用浏览器 `localStorage` 保存数据，包括：

- 打卡记录和打卡规则
- 手动补卡时间、补卡原因和覆盖前记录
- 工作日翻页时刻
- 专注时长和猫咪互动数据
- 休息计时记录
- 久坐提醒记录
- 任务记录
- 手机克制记录
- 请假记录
- 成就数据

注意事项：

- 数据默认只保存在当前浏览器和当前域名/文件来源下。
- 更换浏览器、设备或清空浏览器数据后，历史记录不会自动同步。
- 当前版本保持旧版 `localStorage` key 的兼容写入，便于历史数据继续读取。

## 设计原则

- `本地优先`：不依赖后端，数据留在本地浏览器。
- `静态可运行`：入口 HTML 可直接打开，部署到 GitHub Pages 也可运行。
- `规则可调`：打卡规则、合格规则和工作日翻页时刻都可在界面中调整。
- `科研场景导向`：关注工位、专注、久坐、任务与长期执行反馈。
