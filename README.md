# 研究生工位打卡与时间管理系统

一个面向研究生、科研工作者和个人工位管理场景的轻量级打卡与时间管理系统。

本项目是纯前端静态应用：无需后端、无需数据库，日常使用时直接打开 HTML 文件即可。数据保存在浏览器 `localStorage` 中，适合个人本地长期使用。

## 功能特性

- `打卡记录`：支持上午、下午、晚上三段上班/下班打卡，并自动判断是否合格。
- `规则设置`：支持配置可打卡时间窗、合格时间窗和工作日翻页时刻，保存后立即重算展示。
- `专注时长`：支持自定义倒计时、完成提醒、每日专注统计和猫粮兑换。
- `猫咪陪伴`：根据专注时长兑换猫粮，支持喂食、好感度、表情、台词和点击互动。
- `久坐提醒`：支持 45 分钟坐下 + 5 分钟站立的自动循环，并保存每日站立统计。
- `任务管理`：支持任务开始/结束和任务时长统计。
- `手机克制`：快速记录克制次数，并支持成就反馈。
- `请假记录`：记录请假日期和原因，请假日会影响打卡状态。
- `统计分析`：查看打卡率、时段打卡、任务时长、手机克制和专注时长趋势。

## 快速开始

克隆仓库：

```bash
git clone https://github.com/rqzhang017/Graduate-student-workstation-attendance-system.git
cd Graduate-student-workstation-attendance-system
```

直接用浏览器打开：

```text
研究生工位打卡与时间管理系统.html
```

也可以直接双击该 HTML 文件运行。日常使用不需要启动服务，也不需要运行 `npm`。

## 技术说明

- `HTML + CSS + JavaScript`：纯前端静态运行。
- `localStorage`：所有数据保存在当前浏览器本地。
- `Tailwind CSS`：样式已编译到 `assets/app.css`，不再依赖运行时 Tailwind CDN。
- `Chart.js`：统计页首次打开时按需加载，避免影响普通打卡操作。
- `Font Awesome`：用于基础图标显示。

如果后续修改 Tailwind 源样式，可重新生成 CSS：

```bash
npx tailwindcss -c ./tailwind.config.cjs -i ./assets/tailwind-input.css -o ./assets/app.css --minify
```

## 性能优化

当前版本已经从单 HTML 重构为静态多文件结构，并针对点击响应做了优化：

- 主要业务逻辑迁移到 `assets/app.js`，页面入口更轻。
- 样式迁移到本地静态 CSS，移除 Tailwind 运行时编译开销。
- 点击保存时优先写入主快照，旧版兼容 key 延迟到空闲时写入。
- 高频 DOM 节点使用缓存读取，减少重复查询。
- 专注、任务、久坐倒计时合并为统一秒级 tick，无计时时不运行秒级循环。
- 统计图表只在统计页可见时刷新，Chart 实例复用，不再每次销毁重建。

## 数据存储

系统使用浏览器 `localStorage` 保存数据，包括：

- 打卡记录和打卡规则
- 工作日翻页时刻
- 专注时长和猫咪互动数据
- 久坐提醒记录
- 任务记录
- 手机克制记录
- 请假记录
- 成就数据

注意事项：

- 数据默认只保存在当前浏览器和当前域名/文件来源下。
- 更换浏览器、设备或清空浏览器数据后，历史记录不会自动同步。
- 当前版本保持旧版 `localStorage` key 的兼容写入，便于历史数据继续读取。

## 设计原则

- `本地优先`：不依赖后端，数据留在本地浏览器。
- `静态可运行`：入口 HTML 可直接打开，部署到 GitHub Pages 也可运行。
- `规则可调`：打卡规则、合格规则和工作日翻页时刻都可在界面中调整。
- `科研场景导向`：关注工位、专注、久坐、任务与长期执行反馈。
